# 数据模型说明（迭代 0.0.3 — 管理后台壳）

## 结论

**本迭代无持久化层变更：** 不新增实体、不修改表结构、不要求迁移。

## 沿用模型（概念层）

管理后台壳仅依赖 **既有用户与会话模型**（与 0.0.2 登录一致）：

- **用户**：`User` 实体及对外 DTO `PublicUser`（经 `GET /api/auth/me` 等返回）。
- **会话**：HTTP-only 会话 Cookie + 服务端会话存储（`createUserSession` / `destroySession` 等生命周期，细节以代码为准）。

**本版本不要求**「管理员角色」字段或 RBAC 表；PRD 已说明 RBAC 非本迭代范围。后续若区分「仅管理员可访问 `/admin`」，需在独立迭代中扩展模型或策略，**不在 0.0.3 范围**。
