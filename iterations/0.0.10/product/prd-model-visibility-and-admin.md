# PRD：模型公有/私有与管理后台公有模型（version 0.0.10）

## 文档信息

| 项 | 内容 |
| --- | --- |
| 版本 | `0.0.10` |
| 上游 | `0.0.8` 控制台模型管理、`0.0.9` 账号与偏好 |
| 状态 | 需求快照（与实现 `iterations/0.0.10/backend|frontend` 对齐） |

---

## 1. 背景与目标

### 1.1 背景

- `0.0.8` 已支持登录用户在控制台登记 **私有** 模型接入（Provider、模型名、API Key）。
- `0.0.9` 已支持用户在 **账号与偏好** 中绑定默认对话/向量模型。
- 产品需要：**由运营/管理员统一登记「全站可选用」的公有模型**，普通用户可见、可选用，但 **不可改删**；用户自行登记的仍为 **私有**，仅本人维护。

### 1.2 目标

1. 数据层区分 **`private`（默认）** 与 **`public`（公有）**。
2. **控制台用户** 新建模型均为 **私有**；列表中可看到本人私有 + 全站公有，对公有行 **仅查看**，不可编辑/删除。
3. **管理后台** 提供 **公有模型** 的增删改查；新建即为公有，全站登录用户可在控制台列表与偏好中选用。
4. 运行时选用模型配置时，须校验：当前用户可选用 **「公有」或「本人私有」**（统一入口查询）。

### 1.3 成功标准（验收）

| # | 标准 |
| --- | --- |
| AC1 | 管理后台创建的模型在控制台列表展示为「公有」，普通用户无法编辑/删除。 |
| AC2 | 控制台自建的模型为「私有」，仅创建者可改删。 |
| AC3 | 用户可在 **账号与偏好** 中将默认对话/向量指向某条 **公有** 配置，对话链路能解析到该配置（密钥、Provider 等）。 |
| AC4 | 删除公有模型后，曾指向该配置的用户偏好指针被清空或呈现与既有「失效」策略一致，无悬空不可解析 id。 |

## 2. 功能范围

### 2.1 本期要做（In Scope）

| # | 能力 | 说明 |
| --- | --- | --- |
| F1 | **可见性字段** | `UserModelConfig` 增加 `visibility`：`private` \| `public`；列表 API 返回可区分。 |
| F2 | **控制台列表** | 分页列出「本人私有 **或** 公有」；列展示类型（公有/私有）；公有行操作禁用并提示去管理后台。 |
| F3 | **控制台写操作** | `POST` 仅创建私有；`PATCH`/`DELETE` 仅允许 **本人私有** 记录（公有返回不存在或无权限）。 |
| F4 | **管理后台公有模型** | 独立菜单与页面：列表仅公有、`POST` 创建公有、`PATCH`/`DELETE` 维护公有；删除时清空全站用户对该 id 的偏好指针（事务）。 |
| F5 | **可选用校验** | `findModelConfigUsableByUser`：`id` 命中且 `(public OR userId=当前用户)`；用于 Profile、偏好保存、对话运行时等。 |

### 2.2 可选扩展（若与本版一并落地）

- **预设标签**：模型配置支持可选多选标签（如：免费、文本、视频、声音、嵌入、对话），用于列表与偏好下拉的展示分类；**非本 PRD 必选**，以实现为准。

### 2.3 本期不做（Out of Scope）

- 按组织/租户隔离公有模型（当前为全站可见可选用）。
- 公有模型的审批流、发布版本、灰度。
- 计费与用量统计。

## 3. 用户故事（摘要）

| 角色 | 诉求 |
| --- | --- |
| **普通用户** | 在控制台看到管理员提供的公有模型，并可在偏好中选用；不能误改删公有配置。 |
| **管理员** | 在管理后台维护公有模型列表，与私有模型分流清晰。 |

---

## 4. 依赖与接口约定

- 控制台 API：`/api/console/models`（行为见 `iterations/0.0.10/backend/implementation-notes.md`）。
- 管理后台 API：`/api/admin/model-configs`（管理员鉴权 `withAdminApi`）。
- 类型：`ModelConfigListItem.visibility` 为 `"private" | "public"`。
