## 2. 控制台 · 模型管理（`/console/models`）

### 2.1 信息提示

- 页面上方 **Alert（info）**：说明私有仅本人维护、公有由管理后台维护、全站可选用。

### 2.2 列表（ProTable）

| 列 | 说明 |
| --- | --- |
| **类型** | `public` → 金色 Tag「**公有**」；`private` → 默认色「**私有**」。 |
| **模型名称 / Provider / API Key 掩码 / 更新时间** | 与 `0.0.8` 设计一致；若本版含 **标签** 列，展示为多个小 Tag 或「—」。 |
| **操作** | 公有行：**编辑**、**删除** **禁用**，`title` 提示「公有模型请在管理后台编辑/删除」。私有行：可操作。 |

### 2.3 新建/编辑

- 新建仍为控制台私有模型表单（Provider、模型名、API Key；若有标签表单项则为 **多选预设项**，可选）。
- 编辑仅对私有记录可达（公有在列表已禁用入口）。

---

## 3. 控制台 · 账号与偏好（`/console/profile`）

### 3.1 模型下拉文案

- **对话模型 / 向量模型** 的选项 `label` 建议格式：  
  `[公有|私有] [Provider 展示名] 模型名称`  
  若存在标签：在 **模型名称之后** 追加 ` · 标签1 · 标签2`（与列表语义一致，便于辨认）。

### 3.2 搜索

- 下拉的 `filterOption` 对整段 `label` 做包含匹配（含标签文案）。

---

## 4. 管理后台 · 模型管理（`/admin/models`）

### 4.1 页头与工具栏

- **标题**：建议 **「模型管理（公有）」** 或与菜单一致的短标题；`PageContainer` `ghost` 与 `users` 等页对齐。
- **说明**：文案明确此处维护的是 **全站可选用** 的公有接入。
- **工具栏**：「新建公有模型」+「刷新」。

### 4.2 列表

- 类型列可固定展示「公有」（或省略，因本页仅公有）；其余列与控制台同类信息对齐（模型名、Provider、掩码 Key、更新时间等）。
- 操作：编辑、删除（删除需二次确认）；无「公有/私有」混排时的禁用逻辑（本页仅公有）。

### 4.3 新建/编辑弹窗

- 字段与控制台私有新建类似（Provider、模型名、API Key；可选标签多选）。
- 编辑时 API Key 留空表示不修改（与控制台一致）。

### 4.4 菜单

- 侧栏增加 **模型管理** → `/admin/models`，图标与云/服务器类图标一致（如 `CloudServerOutlined`）。

## 5. 权限与异常

| 场景 | 设计表现 |
| --- | --- |
| 未登录访问管理 API | 跳转登录，`redirect` 含 `/admin/models` 路径（与项目既有约定一致）。 |
| 非管理员 | **403**，提示无管理员权限（前端 `message.error`）。 |
