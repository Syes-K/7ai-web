# 前端验收清单（手工）- version 0.0.2

测试前：执行 `npm run dev`，确保 `data/app.db` 可写（首次会自动建库）。

## 1. 注册（AC-02、AC-04、AC-05、AC-06）

- [ ] 打开 `/register`，可见邮箱、手机号（可选）、昵称、密码、确认密码、验证码与刷新。
- [ ] 不填邮箱提交：浏览器/HTML5 阻止或接口返回校验错误。
- [ ] 密码与确认不一致：接口返回错误提示（或可先加前端校验）。
- [ ] 密码不足 8 位或缺少字母/数字：接口返回策略错误。
- [ ] 手机号填非 11 位：接口报错或输入被限制为数字。
- [ ] 手机号留空：可成功注册（在邮箱等合法前提下）。
- [ ] 验证码错误：提示「验证码错误或已过期…」，刷新后可重新输入。
- [ ] 同一邮箱第二次注册：提示「该邮箱已注册…」。
- [ ] 注册成功：跳转到 `/`（或 URL 中 `?redirect=` 在白名单内时跳转到对应页）。

## 2. 登录（AC-03、AC-05、AC-06）

- [ ] 打开 `/login`，可见邮箱、密码、验证码。
- [ ] 故意错误密码或未注册邮箱：提示「邮箱或密码错误…」（统一文案，不区分是否存在）。
- [ ] 验证码错误：与上不同的错误提示（验证码类）。
- [ ] 登录成功：跳转 `/` 或合法 `redirect`。
- [ ] 访问 `/login?redirect=/chat`，登录成功后落到 `/chat`。

## 3. 受保护路由（AC-07）

- [ ] 未登录直接访问 `/chat`：被重定向到 `/login?redirect=/chat`。
- [ ] 未登录直接访问 `/console`：同上（redirect 含 console）。
- [ ] 登录后再访问 `/chat`、`/console`：可进入（占位内容）。

## 4. 会话与频控（AC-06、AC-08、AC-09）

- [ ] 登录后同浏览器可访问需登录页（与产品一致）。
- [ ] 调用 `POST /api/auth/logout`（可用浏览器控制台 `fetch`）后，再访问 `/chat` 应需重新登录。
- [ ] 短时大量请求验证码/登录：可能出现 429（频控）。

## 5. 视觉与无障碍（抽样）

- [ ] 认证页为深色赛博风，正文与背景对比可读。
- [ ] 表单控件可见 focus 状态。
- [ ] 验证码图片有 `alt` 文案；刷新按钮可点。
