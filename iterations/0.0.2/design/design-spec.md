| 项 | 内容 |

## 2. 设计目标

- 建立可交付、可实现的 **赛博 & 科技黑** 方向：深色底、霓虹点缀、细线/微网格/柔和光晕，避免廉价高饱和闪烁。
- 令牌（色、字、间距、圆角、状态）统一，便于前端用 CSS 变量或主题对象落地。
- 认证区在桌面与移动端均保持**单主视线流**（表单卡片居中或略偏上），背景氛围与前景表单层级分明。

## 3. 视觉语言关键词

| 维度 | 方向 |
| --- | --- |
| 气质 | 冷静、精密、夜间界面；科技感来自结构（线、格、微光）而非复杂插画 |
| 背景 | 近黑渐变 + 极低对比网格或斜向细线纹理（opacity 低，避免干扰阅读） |
| 点缀 | 1～2 种霓虹色用于焦点、主按钮描边/外发光、关键分隔线 |
| 容器 | 半透明深色「玻璃」卡片感可选；边框为 1px 细线 + 微弱内发光 |
| 动效 | 克制：焦点环、按钮 hover 亮度/位移 ≤ 2px、验证码刷新时 150～250ms 淡入；禁止长时间循环强闪烁 |

## 4. 设计令牌（Design Tokens）

### 4.1 色板（语义命名建议）

> 以下为 **HEX 参考**，实现时可映射为 CSS 变量；需满足 WCAG：**正文与背景对比度 ≥ 4.5:1**，大号/粗体次要文案 ≥ 3:1（以实际字号与字重测算为准）。

| Token | 用途 | 参考值 |
| --- | --- | --- |
| `bg.canvas` | 页面最底层背景 | `#050608` → `#0B0F14` 线性渐变（上深下略亮） |
| `bg.elevated` | 认证卡片底色 | `#0E1218` @ 92% 不透明叠于 canvas 上，或实色 `#12161F` |
| `bg.subtle` | 输入框内底、次级区块 | `#151A24` |
| `border.default` | 默认描边 | `rgba(255,255,255,0.08)` |
| `border.focus` | 聚焦环（与霓虹主色协调） | `rgba(0,245,255,0.45)` 或品牌霓虹色 |
| `text.primary` | 主文案 | `#E8EAEF` |
| `text.secondary` | 标签、说明 | `#9AA3B2` |
| `text.disabled` | 禁用态文字 | `#5C6570` |
| `text.on-accent` | 主按钮内文字 | `#050608`（若主按钮为高亮霓虹底）或 `#0A0C10` |
| `accent.primary` | 主霓虹（链接、主按钮、焦点） | `#00E5FF`（青霓虹，可替换为品牌色，全站唯一主点缀） |
| `accent.secondary` | 次要高亮（分隔线、图标） | `#7C4DFF` 或 `#B388FF`（用量少于 primary） |
| `state.danger` | 表单/行内错误 | `#FF5C7A` 或 `#FF6B6B` |
| `state.warning` | 限流、非阻断提示 | `#FFB020` |
| `state.success` | 成功（本版认证页少用，预留） | `#3EE08F` |
| `overlay.scrim` | 全屏加载遮罩 | `rgba(5,6,8,0.55)` |

**霓虹使用约束**：`accent.primary` 用于主 CTA、链接、聚焦；`accent.secondary` 仅用于装饰性分隔或图标，避免两色同时大面积抢戏。

### 4.2 字号与字重层级

| Token | 桌面参考 | 移动参考 | 字重 | 用途 |
| --- | --- | --- | --- | --- |
| `type.brand` | 20–22px | 18–20px | 600 | 顶栏产品名 / 认证区上方标识 |
| `type.title` | 24–28px | 22–24px | 600 | 页面主标题（登录 / 注册） |
| `type.body` | 15–16px | 15px | 400–450 | 表单标签、输入内容、正文提示 |
| `type.small` | 13–14px | 13px | 400 | 辅助说明、底部协议、验证码旁提示 |
| `type.caption` | 12px | 12px | 400 | 错误行内辅助、限流倒计时旁注 |

行高：标题 1.25～1.3；正文与输入 1.45～1.55。

### 4.3 间距（8pt 基准）

| Token | 值 | 用途 |
| --- | --- | --- |
| `space.xs` | 4px | 图标与文字、行内元素 |
| `space.sm` | 8px | 标签与输入、紧凑堆叠 |
| `space.md` | 16px | 表单项之间默认间距 |
| `space.lg` | 24px | 表单组与标题、卡片内边距（竖向） |
| `space.xl` | 32px | 卡片外与视口边、区块分隔 |

认证卡片内边距：桌面 `24–32px`，移动 `20–24px`。

### 4.4 圆角与描边

| Token | 建议值 | 用途 |
| --- | --- | --- |
| `radius.card` | 12–16px | 认证主卡片 |
| `radius.control` | 8–10px | 输入框、按钮、验证码图片容器 |
| `radius.pill` | 9999px | 可选：次要标签按钮 |
| `border.hairline` | 1px | 默认输入框、卡片外框 |

### 4.5 阴影与光晕

- **卡片**：`0 0 0 1px rgba(255,255,255,0.06)` + `0 24px 48px rgba(0,0,0,0.45)`，可选底部 `0 0 40px rgba(0,229,255,0.06)` 极弱环境光。
- **主按钮（默认）**：轻微 `box-shadow` 同色系外扩；**hover** 增加亮度与 1 级外发光，**active** 略压暗。
- **危险/错误条**：左侧 3px 实色条或整行浅红底 `rgba(255,92,122,0.12)`，保证与 `text.primary` 对比可读。

## 5. 认证区全局布局

### 5.1 页面结构（两页共用骨架）

1. **氛围层（全屏）**：`bg.canvas` + 可选 CSS/SVG 网格或斜线（低透明度）+ 径向渐变光斑（顶部偏一侧，避免遮挡表单）。
2. **顶栏（可选但推荐）**：左侧产品名/Logo 字标（`type.brand`）；右侧不放登录/注册切换（切换放在卡片内更清晰）。高度约 56–64px，与内容区间距 `space.lg`～`space.xl`。
3. **主内容区**：单列；认证卡片 `max-width` 建议 **420–480px**（桌面），移动为视口宽减去左右各 `16px`。
4. **页脚辅助区（可选）**：版权或极简链接，`type.small` + `text.secondary`。

### 5.2 垂直节奏

- 卡片内自上而下：**标题** → **一句副文案（可选）** → **表单** → **主按钮** → **次要链（注册/登录互链）**。
- 首屏保证 **主按钮与互链** 在常见视口高度内可见（移动优先），长表单（注册）在移动上允许自然滚动。

### 5.3 响应式断点（与工程可对齐）

| 名称 | 范围 | 调整 |
| --- | --- | --- |
| Mobile | `< 768px` | 卡片全宽（减边距）；标题字号降一档；表单项间距保持 `space.md` |
| Tablet | `768px–1199px` | 卡片居中，`max-width` 同上 |
| Desktop | `≥ 1200px` | 可适当增大背景光斑与卡片 `max-width` 至 480px |

## 6. 组件级视觉约定（认证上下文）

### 6.1 文本输入

- 高度：触摸友好 **44–48px**（含内边距）。
- 默认：`bg.subtle` + `border.default`；**hover** 略亮边框；**focus**：`border.focus` + 外环 2px 半透明（与 `accent.primary` 一致）。
- **error**：边框 `state.danger` 或左侧竖条 + 下方 `type.caption` 错误文案。
- **disabled**：`bg.subtle` 降对比 + `text.disabled`，`cursor: not-allowed`。

### 6.2 主按钮（提交）

- 默认：霓虹渐变或实色 `accent.primary` 底 + `text.on-accent`；**loading**：文案区换为 spinner +「登录中/注册中…」，按钮禁用点击。
- **disabled**（表单未通过客户端校验时）：降低不透明度至约 **0.45–0.55**，仍可见文案，避免与「可点」混淆。

### 6.3 次要链接（互链、忘记类占位）

- 样式：`accent.primary` 下划线可选；**hover** 提亮；**focus** 可见焦点环（与 FR-AUTH-003 互链可见性一致）。

### 6.4 图形验证码区域（外观级）

- **图片容器**：固定最小高度 **40–44px**，宽度约 **100–120px**（比例按后端图）；`radius.control` + `border.default`，内底略深以衬托字符。
- **刷新**：图标按钮（循环箭头）附在图片右侧或下方，**最小点击区域 44×44px**；**hover** 霓虹描边；**loading 刷新中** 图片区 opacity 降低 + 小 spinner 覆盖。
- 与输入框同一「表单项」分组，间距 `space.sm`～`space.md`。

## 7. 状态总览（全局）

| 状态 | 视觉要点 |
| --- | --- |
| 默认 | 见 §6 |
| 加载（提交中） | 主按钮内 spinner；可选全屏/卡片 `overlay.scrim` 防重复提交（产品未强制，由实现择一） |
| 成功 | 短暂成功反馈后可立即跳转（见 FR-AUTH-006）；本版可不单独做 toast，避免双反馈 |
| 客户端校验错误 | 字段下红字或 `state.danger` 条；不抖动整页 |
| 服务端错误 | 卡片顶部 **Alert 条**（可关闭）或 **单行汇总** + 必要时字段级 |
| 限流 | 独立文案色 `state.warning`；可配倒计时 `type.caption` |
| 禁用 | 整表单或单控件见 §6 |

## 8. 无障碍（A11y）原则

- **对比度**：霓虹仅用于大按钮、链接与粗标题时需注意与深底对比；正文与标签仅用 `text.primary` / `text.secondary` 已校准色。
- **焦点**：所有可聚焦控件可见焦点环（与 `border.focus` 一致）；Tab 顺序与视觉顺序一致（标题 → 字段 → 验证码 → 刷新 → 提交 → 互链）。
- **表单标签**：每个输入与验证码输入均有可见 `<label>` 或 `aria-label`（实现阶段）。
- **错误关联**：`aria-describedby` 指向错误文案容器（实现阶段）。
- **验证码替代**：纯图片验证码对视障用户不友好；**原则**：后续工程可评估「音频验证码 / 人机挑战替代方案」等（不在本版需求范围）；设计侧预留验证码区块旁 **辅助说明文案位**（见 `spec-auth-pages.md`），不承诺本版功能。
