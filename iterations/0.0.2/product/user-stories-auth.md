# User Stories - 认证（登录 / 注册）- version 0.0.2

## 使用说明

本文件为 `prd.md`、`requirements.md` 的补充，按角色拆解用户故事与验收标准（AC）。验收测试可直接映射至下列 AC。

---

## US-01 新用户注册

**作为** 新用户  
**我想要** 使用**必填邮箱**、可选手机号、昵称和密码完成注册，并在提交前完成图形验证码  
**以便** 获得账号并直接进入可使用产品的已登录状态

**验收标准：**

- [ ] **AC-US-01-1** 我能打开注册页，看到**邮箱（必填）**、**手机号（可选）**、昵称、密码、确认密码及**图形验证码**（标签文案可因设计而异，但语义须对应）。
- [ ] **AC-US-01-2** 当我未填邮箱或邮箱格式明显无效时，无法完成成功注册，并能看到格式相关提示。
- [ ] **AC-US-01-3** 当我填写手机号时，若非 11 位纯数字，无法完成成功注册或须看到格式提示；我可以在不填手机号的情况下继续（若其他字段合法）。
- [ ] **AC-US-01-4** 当两次密码不一致时，无法完成成功注册，并能看到一致性问题提示。
- [ ] **AC-US-01-5** 当邮箱已被注册时，注册失败，并能看到「已注册」类提示及前往登录的引导。
- [ ] **AC-US-01-6** 当图形验证码输入错误时，我得到明确提示并可刷新验证码后重试。
- [ ] **AC-US-01-7** 注册成功后，我处于已登录状态，并被带到默认落地页（首页 `/`），或符合 `redirect` 规则的目标页。

---

## US-02 老用户登录

**作为** 已注册用户  
**我想要** 使用**邮箱**与密码登录，并在提交前完成图形验证码  
**以便** 继续访问首页、对话与控制台等能力

**验收标准：**

- [ ] **AC-US-02-1** 我能打开登录页，输入**邮箱**、密码、**图形验证码**并提交。
- [ ] **AC-US-02-2** 当邮箱或密码错误时，登录失败；提示策略符合 `requirements.md` FR-AUTH-005（不泄露邮箱是否已注册）。
- [ ] **AC-US-02-3** 当图形验证码错误时，登录失败或须先修正验证码，行为可测且可与凭证错误区分（具体由实现约定，须不误导用户）。
- [ ] **AC-US-02-4** 登录成功后，我处于已登录状态，并被带到默认落地页（`/`）或合法 `redirect` 目标。
- [ ] **AC-US-02-5** 我能在登录页找到前往注册页的入口。

---

## US-03 错误与限制

**作为** 用户  
**我想要** 在出错或受限时知道发生了什么以及能否重试  
**以便** 我能修正输入或择时再次尝试

**验收标准：**

- [ ] **AC-US-03-1** 频控触发时，我能看到「请求过于频繁/请稍后再试」类提示，且仍能理解需要等待而非系统崩溃。
- [ ] **AC-US-03-2** 若账号被禁用或不可用，我能看到不可用说明（若业务提供该状态）。
- [ ] **AC-US-03-3** 密码不符合长度或字母+数字策略时，在注册或相关校验路径上能被明确提示（以 `requirements.md` 密码策略为准）。

---

## US-04 会话与页面可达性

**作为** 已登录用户  
**我想要** 在登录后访问首页、对话页、控制台而不被无故当作未登录  
**以便** 我能连续使用产品

**验收标准：**

- [ ] **AC-US-04-1** 登录成功后，我能访问 `/`、`/chat`、`/console`（在仅认证、无额外业务权限前提下）。
- [ ] **AC-US-04-2** 未登录时访问 `/chat`、`/console`，我会被引导至登录流程或等价拦截，且看不到需登录才应有的业务数据。

---

## US-05 登出

**作为** 已登录用户  
**我想要** 登出  
**以便** 结束本会话，他人共用设备时更安全

**验收标准：**

- [ ] **AC-US-05-1** 我能通过界面上的某一处入口执行登出（具体位置由设计定）。
- [ ] **AC-US-05-2** 登出后，再访问 `/chat`、`/console` 时，需重新登录才能进入已登录态下的体验。

---

## Out of Scope（本文件不验收）

- 短信 OTP、**邮箱一次性数字验证码**、邮箱验证链接。
- 第三方登录与 SSO。
- 找回密码、换绑手机、注销账号。
- 控制台与对话的具体业务功能。
