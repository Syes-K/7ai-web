# 迭代说明（version 0.1.12）

本迭代围绕“公安备案场景下的测试账户”展开，实现目标是让访客可体验聊天页面，同时确保测试账户不具备数据写入能力。

## 文档索引

- 产品需求：`iterations/0.1.12/product/prd.md`
- 后端实现：`iterations/0.1.12/backend/implementation-notes.md`
- 前端实现：`iterations/0.1.12/frontend/implementation-notes.md`

## 本轮关键决策

1. 全局仍保留后端只读写请求拦截，作为安全兜底。
2. 聊天发送接口因提示体验需要差异化处理，改为前端发送入口先行拦截并提示。
3. 管理端用户管理提供只读开关，支持按账号动态控制测试权限。
4. 登录与退出属于认证流程，不纳入只读写拦截（避免误拦截体验）。
5. 首页补充备案信息与作者联系方式，登录页补充测试账号使用说明。
