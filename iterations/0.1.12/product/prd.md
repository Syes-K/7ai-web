| 状态 | 已完成实现并进入联调验收 |

## 背景与目标

为满足公安备案场景，系统不再对外开放直接注册。同时为了让访客可体验对话页面，需要提供“可登录但不可修改数据”的测试账号能力。

本轮目标：

1. 访客打开登录页即可快速体验测试账户登录。
2. 管理员可在用户管理中将指定账户切换为测试账户（只读）。
3. 测试账户登录后只能进行 `GET` 请求，写操作被限制并返回明确提示。
4. 聊天发送因交互提示需与其他接口区分，在前端发送入口先行拦截并提示。

## 需求条目

1. 登录页默认填充测试账户：
   - 账号：`test@7ai.club`
   - 密码：`test1234`
2. 管理后台用户管理提供测试账户（只读）开关（可开启/关闭）。
3. 测试账户访问 API 时：

   - 拒绝 `POST` / `PUT` / `PATCH` / `DELETE` 等写请求
   - 提示统一文案：`您访问的是测试账户，不能进行数据的修改和删除`
4. 聊天页发送能力（按钮发送、回车发送、失败重试）：
   - 测试账户在前端直接拦截，不发起发送请求
   - 提示文案：`测试账户仅支持浏览，不支持发送消息`

## 验收要点

1. 登录页初始渲染后，账号和密码输入框已带默认值。
2. 管理员可在用户列表中看到“访问模式（读写/只读）”并可切换。
3. 测试账户发起非聊天写请求时返回 `403 FORBIDDEN`，且前端可读取统一错误文案。
4. 测试账户在聊天页触发发送/重试时，前端即时提示“测试账户仅支持浏览，不支持发送消息”，且网络层不发起发送接口调用。
5. 首页底部可见备案号与邮箱信息，且链接可点击（备案跳转工信部、邮箱可发邮件）。
6. 登录页底部展示新的测试账号说明文案，视觉层级低于主操作区。
7. 非只读用户行为不受影响，现有业务流程可正常写入数据。
