# 手动验收清单：对话页助手绑定（0.1.2）

测试前：已登录，可访问 `/chat`；若测「可选用助手」列表，需在控制台至少零个或多个可见助手。

## PRD §3.1 新建会话前可选助手 / 跳过

| 步骤 | 预期 |
| --- | --- |
| 打开「新建对话」Modal | 出现说明文案；默认不预选助手。 |
| 在助手列表加载过程中点击「跳过 · 普通对话」 | 可创建**无绑定**会话并进入主区（不必等列表结束）。 |
| 列表加载完成后选中助手，点「开始对话」 | 创建**已绑定**会话；侧栏新项带助手 emoji+名称；首条为开场白或默认问候（见 §3.2）。 |
| 点遮罩或关闭 Modal | 不创建新会话，当前选中会话不变。 |

## PRD §3.2 开场白 / 空开场默认

| 步骤 | 预期 |
| --- | --- |
| 绑定「有开场白」的助手新建 | 首条助手消息与配置一致。 |
| 绑定「开场白为空」的助手新建 | 首条为产品固定问候（与后端常量/设计 §6.1 一致）。 |
| 普通对话新建 | 无首条助手开场注入（与既有行为一致）。 |

## PRD §3.3 / §3.4 身份展示与历史列表

| 步骤 | 预期 |
| --- | --- |
| 已绑定会话 | 助手消息气泡旁为 **emoji + 名称**；侧栏同会话展示助手标识 + 标题 + 最后活动时间；**无**预览副标题行。 |
| 普通对话 | 助手消息为元信息 **「助手」**（泛化）；侧栏无助手 chip。 |
| 助手已删除但会话仍存在（`assistantUnavailable`） | 侧栏为 **「助手不可用」** 占位；主区顶部 **警告条**；助手消息元信息为 **「助手」**；可继续发消息、可新建对话。 |

## 设计 §3.1～§3.3 Modal 异常与空列表

| 步骤 | 预期 |
| --- | --- |
| 模拟助手列表接口失败（如断网后打开 Modal） | 内联错误提示 + **重试**；**跳过**仍可用；**开始对话**在加载中禁用。 |
| 可选用助手为空 | 空态文案；**跳过**可用；**开始对话**禁用；存在链至 **`/console/assistants`**（有控制台权限时可用）。 |

## PRD §3.2 AC4 清空（与后端方案 A）

| 步骤 | 预期 |
| --- | --- |
| 清空已绑定助手会话 | 消息清空；**不**再次自动注入开场白；会话仍存在列表。 |

## 回归：防串会话与发送

| 步骤 | 预期 |
| --- | --- |
| 切换会话 | 主区加载态正确；消息与当前会话一致。 |
| 发送消息 / 流式 | 与 0.0.6 一致；绑定与否不改变失败重试策略。 |
