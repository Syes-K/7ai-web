## 2. 安全风险

| 风险 | 缓解 |
|------|------|
| DB 泄露导致密钥外泄 | 推荐应用层加密 + 环境密钥；限制 DB 文件权限；备份加密（运维面）。 |
| 日志打印明文 Key | 代码审查与统一脱敏；禁止在 `catch` 中 dump 整个 body。 |
| IDOR | 所有 PATCH/GET-by-id 必须校验 `userId`；他人资源返回 **404**（不 403 暴露存在性，可选策略以安全规范为准）。 |

---

## 3. 产品/设计已收敛的假设（回溯）

以下来自设计与 PRD，后端实现须遵守：

- **`DELETE`**：物理删除，须校验 `userId`，他人资源 **404**（与 PATCH 一致）。  
- 编辑可改 `provider`。  
- 列表与接口不返回明文 Key；编辑 **留空不改密钥**（省略或空字符串）。  

若产品变更（例如改为软删除），需回写 PRD/设计并更新 `api-spec.md`、`data-models.md`。
