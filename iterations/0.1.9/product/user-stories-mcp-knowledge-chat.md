# 用户故事与验收标准：MCP × 知识库 × 对话（version 0.1.9）

本文档为 `prd.md` 的子文档，便于评审与测试用例编号。

---

## Epic A：个人控制台 — MCP 管理

### US-A1 管理我的 MCP 配置列表

**作为** 已登录用户  
**我想要** 在控制台查看我名下所有 MCP 配置的列表（名称、启用状态、最近连接检测结果）  
**以便** 快速了解我的外部能力接入情况并进行维护  

**验收标准：**

- [ ] **AC-M1**：未登录访问 `/console/mcp` 被重定向到登录页；匿名调用相关 API 返回 401。
- [ ] **AC-M2**：仅返回当前 `userId` 下的配置行；切换账号后列表内容随之变化。
- [ ] 列表支持按名称关键字过滤（可与知识库列表交互复杂度对齐）。

### US-A2 新建与编辑 MCP 配置

**作为** 已登录用户  
**我想要** 新建或编辑一条 MCP 配置（名称、连接参数、启用开关）  
**以便** 在知识库与对话中使用稳定的 MCP 引用  

**验收标准：**

- [ ] **AC-M3**：必填项缺失时保存失败并提示字段级错误；成功保存后再次进入编辑页字段与库中一致。
- [ ] 密钥/Token 类字段：创建后再次打开编辑页时 **不以明文展示完整值**（占位符或「已配置」态）；支持「更新密钥」独立操作（具体交互由设计定稿）。
- [ ] `enabled = false` 时，该配置 **不得** 被对话运行时加载为可用 MCP（知识库若仍挂载则行为见 US-K3 / 开放问题）。

### US-A3 测试 MCP 连接

**作为** 已登录用户  
**我想要** 对某条 MCP 配置执行「测试连接」  
**以便** 在挂载到知识库前确认网络与凭证可用  

**验收标准：**

- [ ] **AC-M4**：测试完成后列表或详情中可见最近检测时间、成功/失败状态；失败时展示 **已脱敏** 的错误摘要（不暴露密钥、不返回原始堆栈给用户）。
- [ ] 测试过程中有明确 loading，可重复触发。

### US-A4 删除 MCP 配置

**作为** 已登录用户  
**我想要** 删除不再使用的 MCP 配置  
**以便** 减少暴露面与列表噪音  

**验收标准：**

- [ ] 若该配置仍被至少一个知识库挂载：行为与产品定稿一致（**阻止删除并提示** 或 **级联解除挂载**，见 `README.md` 开放问题）。
- [ ] 删除成功后知识库侧不再显示该挂载。

## Epic B：知识库 — 多 MCP 挂载

### US-B1 为知识库挂载多个 MCP

**作为** 已登录用户  
**我想要** 在某个知识库的编辑/详情界面多选挂载多个我的 MCP 配置  
**以便** 该知识库相关的对话能同时使用多个外部工具源  

**验收标准：**

- [ ] **AC-K1**：同一知识库可选择 0~N（N 上限若有限制需在开放问题确认）个 MCP；保存后重新打开回显一致。
- [ ] 选择器仅展示当前用户已创建的配置；禁用项展示规则与 `enabled` 一致（见开放问题）。

### US-B2 挂载权限与隔离

**作为** 平台  
**我想要** 保证用户只能挂载自己的 MCP 到自己的知识库  
**以便** 满足数据隔离与合规  

**验收标准：**

- [ ] **AC-K2**：伪造他人 MCP id 写入知识库挂载时，服务端拒绝（422/403 等统一策略），且不泄露资源是否存在（策略与全局错误规范对齐）。

### US-B3 禁用或删除 MCP 后的挂载一致性

**作为** 已登录用户  
**我想要** 在禁用或删除 MCP 后，知识库不会出现无效引用或静默失败  
**以便** 我能够修正配置并得到明确反馈  

**验收标准：**

- [ ] **AC-K3**：符合产品在「删除/禁用」上的定稿策略（见 `README.md`）；至少保证不会出现跨用户数据引用。

## Epic C：对话 — Agent 运行时挂载 MCP

### US-C1 助手会话合并知识库上的 MCP

**作为** 使用绑定了知识库的助手的用户  
**我想要** 在对话时 Agent 自动具备这些知识库所挂载 MCP 对应的工具能力  
**以便** 我无需在每次对话中重复配置  

**验收标准：**

- [ ] **AC-C1**：给定助手绑定知识库 K1、K2；K1 挂载 MCP-a，K2 挂载 MCP-b；发送用户消息时，构建 Agent 的 tools 解析路径包含来自 a、b 的工具（去重规则：同一 MCP id 只加载一次）。
- [ ] 实现路径须经过现有 `resolveAllToolsForAgent`（或其后继统一入口），不得在路由层旁路拼装第二套 Agent。

### US-C2 无知识库或无 MCP 时兼容

**作为** 使用未绑定知识库的助手或纯默认对话的用户  
**我想要** 对话行为与改造前一致  
**以便** 不受影响  

**验收标准：**

- [ ] **AC-C2**：助手 `knowledgeBaseIds` 为空，或所有绑定知识库均未挂载 MCP 时，不产生 MCP tools；主流程无额外错误。

### US-C3 部分 MCP 失败降级

**作为** 用户  
**我想要** 在某一 MCP 不可用时仍能收到模型回复  
**以便** 核心业务（对话）不被单点外部故障完全阻断  

**验收标准：**

- [ ] **AC-C3**：模拟 MCP-a 失败、MCP-b 成功：本轮 Agent 仍可用 b 的工具；助手正文仍可生成。
- [ ] 全部 MCP 失败时退化为无 MCP tools；不要求用户刷新页面即可继续对话。
