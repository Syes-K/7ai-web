## 1. 背景与问题陈述

用户在对话与知识库场景中需要接入 **MCP（Model Context Protocol）** 能力：一方面希望在**个人控制台**集中管理自己的 MCP 服务端点与凭证；另一方面希望**知识库**不仅能承载静态可检索内容，还能声明依赖哪些 MCP（例如文档库外的实时数据源）；最后在**对话运行时**，当会话实际使用到绑定了 MCP 的知识库时，这些 MCP 应作为 **LangChain Agent 的 tools 来源**与现有对话链路一致，而非仅停留在配置层。

当前仓库内已有：

- **控制台**：`/console` 下 ProLayout 壳，侧栏含「账号与偏好」「模型管理」「助手管理」「知识库管理」等（`src/app/console/console-menu.tsx`）。
- **知识库**：`KnowledgeBase` 实体与 `/api/knowledge-bases` 系列接口；助手与知识库为多选绑定（见迭代 `0.1.6` 文档与 `PUT /api/console/assistants/:id/knowledge-bases`）。
- **对话**：`getAssistantAgent` 经 `resolveAllToolsForAgent` 合并原生工具与 MCP 衍生工具；`turn-capabilities.ts` 中 **`loadMcpBindingsForChatTurn` / `mcpBindingsToLangChainTools` 当前为空实现**，已预留 `McpServerBinding` 类型与扩展注释。

本需求在上述能力上增加「用户级 MCP 配置」「知识库 ↔ MCP 多对多关系」「按会话/助手上下文将 MCP 注入对话 Agent」的产品定义，供设计与研发对齐。

---

## 2. 目标（Goals）

1. **可治理**：登录用户可在个人控制台完成 MCP 相关配置的增删改查、连接校验（或等价健康检查）、启用/停用，且仅影响当前用户数据。
2. **可组合**：单个知识库可挂载 **0 个或多个** MCP 引用（来自同一用户已配置的 MCP），关系清晰、可审计。
3. **可运行时一致**：当本轮对话在业务上「使用了绑定了 MCP 的知识库」时，对应 MCP 必须在 **与主对话相同的 Agent 构建路径**中参与 tools 解析（与 `resolveAllToolsForAgent` 语义一致），并定义失败时的降级与对用户可见的反馈边界。
4. **可验收**：功能范围、数据隔离、权限与 AC 可测试；与「无知识库 / 无 MCP / MCP 失败」场景兼容。

---

## 3. 非目标（Non-Goals）

- 不在本 PRD 中规定具体 MCP Server 清单、第三方 MCP 市场或计费模型。
- 不要求本期实现「团队/组织级共享 MCP」或跨用户复用（除非后续明确开放；默认 **用户私有**）。
- 不替代知识库向量检索：MCP 作为 **工具能力** 补充，不强制与「是否命中检索片段」绑定（具体策略见第 6 节「待产品确认」可二选一）。
- 不在此阶段锁定具体传输协议实现细节（stdio / SSE / HTTP）；产品层定义「连接配置 + 可用性校验 + 运行时绑定」行为，协议选型可作为研发设计项。

## 4. 用户与核心场景

| 角色 | 说明 |
| --- | --- |
| 普通登录用户 | 拥有个人控制台、个人知识库、个人助手与对话会话。 |

**核心场景摘要：**

1. 用户在控制台新增一个 MCP 配置，填写必要字段，保存后执行「连接测试」，看到成功/失败与可读错误摘要。
2. 用户在知识库详情或编辑流程中，从「我的 MCP」列表多选挂载；保存后列表展示已挂载项，可移除。
3. 用户绑定助手到若干知识库后开始对话；若这些知识库挂载了 MCP，则对话中 Agent 可调用的工具集合包含这些 MCP 映射出的工具（在实现上对接现有 `loadMcpBindingsForChatTurn` 链路）。

详细用户故事与验收标准见同目录 **`user-stories-mcp-knowledge-chat.md`**。

## 5. 功能范围

### 5.1 个人控制台 — MCP 管理

**定义「用户 MCP 配置」**（名称可研发展示为「MCP 连接」或「MCP 服务」，产品统一称「MCP 配置」）为一条用户私有记录，建议包含字段（研发可微调命名与存储形态，**语义不得缺失**）：

| 字段组 | 建议字段 | 说明 |
| --- | --- | --- |
| 标识 | `id`、用户外键 | 全局唯一 id；**必须**归属 `userId`，所有读写经用户鉴权过滤。 |
| 展示 | `name`、`description?` | 短名称用于列表与知识库挂载选择器；描述可选。 |
| 连接 | `transport`（枚举占位）、`endpoint` 或等价结构、`metadata`（JSON 扩展） | 足以描述「如何连上 MCP」；敏感信息见下栏。 |
| 安全 | `credentials` 或引用密钥库 | **不落日志、不明文回显**；编辑时可「置空表示不修改」策略需在交互上定义。 |
| 状态 | `enabled`、`lastCheckedAt`、`lastCheckStatus`、`lastErrorSummary?` | 支持手动「测试连接」与可选周期性校验（周期是否做进本期见开放问题）。 |

**控制台能力：**

- 侧栏新增入口（与现有「知识库管理」同级），路径建议 `/console/mcp`（最终以设计稿为准）。
- 列表：搜索/筛选（至少按名称）、启用状态、最近检测结果。
- 新建/编辑：表单校验、保存、删除（删除前校验是否仍被知识库引用；若被引用则 **禁止硬删** 或 **强制先解除挂载** — 二选一，见开放问题）。
- 测试连接：异步或同步 + loading；结果写入 `lastCheck*` 字段。

### 5.2 知识库 — 挂载多个 MCP

**关系模型（逻辑）：**

- **多对多**：`KnowledgeBase` : `UserMcpConfig` = N : M，通过中间表（或等价 JSON 数组，**推荐中间表** 以便约束与查询）表达。
- **约束**：仅允许挂载 **同一 `userId`** 下的 MCP 配置；挂载记录不包含其他用户 id。
- **配置入口**：在控制台「知识库管理」新建/编辑或详情中增加「MCP 挂载」区块（多选、展示已选、跳转管理 MCP 的链接）。

**权限与数据隔离：**

- 列表/读写知识库与挂载关系时，一律以会话用户 `userId` 为边界；禁止通过 id 遍历他人资源。
- 删除 MCP 配置时，必须保证不会出现「孤儿挂载」或需级联策略（见开放问题）。

### 5.3 对话流程 — MCP 挂载到对话 Agent

**触发条件（产品默认提案，可经开放问题定稿）：**

- 会话存在 **`assistantId`**（即非「纯默认系统提示」的助手会话），且该助手已绑定一个或多个知识库；**取这些知识库的 MCP 挂载并集**，按 `McpConfig.id` 去重，得到本轮 `McpServerBinding[]`（或与现有 `McpServerBinding` 对齐的 id 列表）。
- **未绑定助手的会话**：若当前产品形态下对话 **从不** 单独选择知识库，则本期 **不要求** 注入知识库 MCP；若未来支持「会话级知识库」，则另起需求扩展 `ChatTurnCapabilityContext`（本 PRD 标注为衔接假设）。

**与知识库检索的关系：**

- **默认提案 A（推荐）**：只要助手绑定的知识库集合非空且其中存在 MCP 挂载，则 **每一轮** Agent 构建均合并这些 MCP tools，**不依赖** 本轮是否执行向量检索或是否命中片段（工具可用性与 RAG 解耦）。
- **备选提案 B**：仅当本轮执行了知识库检索且意图判定需要外部能力时才挂载 MCP（实现复杂、用户难理解，仅当合规/成本强烈要求时采用）。

**失败降级：**

- 某一 MCP 连接失败（超时、鉴权失败、协议错误）：该 MCP 对应 tools **从本 turn 剔除**；其余 MCP 与原生 tools **仍可用**。
- 若 **全部** MCP 失败：等价于当前「无 MCP」行为；可选向用户展示一次性系统提示或 UI 角标（**不阻塞** 主模型文本生成）。
- 记录：服务端结构化日志 + 若有工具追踪管道则进入现有 `ToolTrace`（与 `assistant.ts` 中 `onToolEvent` 方向一致，**待研发确认**）。

**与无知识库场景兼容：**

- 助手未绑定知识库，或知识库均未挂载 MCP：`loadMcpBindingsForChatTurn` 行为与今日一致（空列表），对话功能不变。

## 6. 验收标准（汇总）

以下与 `user-stories-mcp-knowledge-chat.md` 中 AC 编号对应，便于测试用例拆分。

| 编号 | 验收摘要 |
| --- | --- |
| AC-M1 | 非登录用户无法访问 MCP 管理 API 与页面。 |
| AC-M2 | 用户 A 无法读取/修改用户 B 的 MCP 配置与测试连接结果。 |
| AC-M3 | MCP 配置保存后列表可见；编辑后刷新字段正确；密钥类字段不明文回显。 |
| AC-M4 | 「测试连接」返回明确成功/失败态并落库可查询字段。 |
| AC-K1 | 知识库可挂载 0~N 个 MCP，多选保存后回显一致。 |
| AC-K2 | 不可挂载其他用户的 MCP id（服务端拒绝，错误信息不泄露存在性细节或遵循统一错误策略）。 |
| AC-K3 | 删除或禁用 MCP 后的挂载一致性符合选定策略（禁止删 / 级联解除 / 阻止禁用）。 |
| AC-C1 | 助手绑定含 MCP 的知识库时，对话请求路径下 Agent 使用的 tools 包含 MCP 衍生工具（可通过集成测试或 mock MCP 断言 `resolveAllToolsForAgent` 入参来源）。 |
| AC-C2 | 助手未绑定知识库或无 MCP 时，行为与改造前一致。 |
| AC-C3 | 部分 MCP 失败时主对话仍返回助手正文；失败 MCP 不产生可用 tools 或调用被短路。 |

## 7. 与现有实现的衔接点（研发对照）

| 能力 | 假设/现状 | 本期衔接建议 |
| --- | --- | --- |
| 控制台导航 | `consoleMenuRoutes` 静态配置 | 增加 MCP 菜单项与 `page.tsx`。 |
| 知识库实体 | `KnowledgeBase` 无 MCP 字段 | 新增关联表或字段；API `PATCH/GET` 知识库时扩展 `mcpConfigIds` 或子资源。 |
| 助手-知识库 | 已有多选绑定 API | MCP 挂载在知识库侧聚合；对话侧按助手 → 知识库 → MCP 解析。 |
| Agent tools | `turn-capabilities.ts` | 实现 `loadMcpBindingsForChatTurn`：输入扩展为可携带 `assistantId` 与会话 id（若需）；从 DB 解析绑定。 |
| 发消息管道 | `post-message-pipeline` / `buildKnowledgeInjectionForChat` | KB 注入与 MCP tools 并行准备；顺序以设计为准，**不得**绕过 `getAssistantAgent`。 |

若数据模型或 API 与上表冲突，以阶段 2 设计文档为准，并回写本 PRD「修订记录」。

## 8. 依赖与风险

| 类型 | 说明 |
| --- | --- |
| 依赖 | 用户登录与会话模型不变；助手与知识库绑定能力已存在。 |
| 安全 | MCP 可能带出网访问用户配置 endpoint；需限流、超时、SSRF 防护策略（研发方案）。 |
| 成本 | 工具调用可能增加 token 与外部调用次数；可配置每 turn 最大 MCP 数（开放问题）。 |
| 体验 | MCP 工具过多可能导致模型选择困难；可后续做「按知识库分组展示」或优先级（非本期必须）。 |

## 10. 开放问题（待产品确认）

以下与 `README.md` 同步；定稿后请在两处以「修订记录」或勾选方式更新。

1. **MCP 挂载与 RAG 是否解耦**：采用「每轮均挂载助手绑定知识库上的 MCP（提案 A）」还是「仅在某条件下挂载（提案 B）」。
2. **每用户 / 每知识库 MCP 数量上限**：是否需要硬限制。
3. **删除仍被引用的 MCP 配置**：禁止删除 vs 允许删除并自动解除所有挂载（及审计要求）。
4. **`enabled = false` 且仍被知识库挂载**：对话忽略并警告、禁止保存知识库、或自动剔除挂载。
5. **连接测试**：仅手动 vs 增加定时探测（通常非首版）。
6. **非助手会话**：本期是否明确不支持「会话级知识库 + MCP」；若未来支持，是否单独立项。
7. **失败与加载态对用户可见级别**：仅服务端日志 vs Toast vs 消息脚注等。

更多上下文与文件索引见 **`README.md`**。
