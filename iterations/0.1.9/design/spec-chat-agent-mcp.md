# 规格：对话 — Agent 运行时 MCP 与用户感知（version 0.1.9）

**相关代码位置（研发对照，设计不改代码）**：  
`turn-capabilities.ts`（`loadMcpBindingsForChatTurn`）、`getAssistantAgent` / `resolveAllToolsForAgent`、`src/components/chat/ChatWorkspace.tsx` 等对话主路径。

---

## 1. 业务触发条件（与 PRD 5.3 对齐）

- **前提**：会话绑定 **`assistantId`**（助手会话），且该助手已配置一个或多个知识库。
- **工具集合并规则**：取这些知识库各自挂载的 MCP 配置 id，做 **并集**，再按 `McpConfig.id` **去重**，得到本轮参与工具解析的 MCP 集合（与 PRD「助手+知识库 → MCP 并集」一致）。

---

## 2. 与 RAG / 向量检索的关系（设计默认假设）

| 产品选项 | 设计说明 |
| --- | --- |
| **提案 A（本设计默认假设）** | MCP tools 的加载与 **本轮是否执行向量检索、是否命中 chunk** **解耦**。只要助手绑定的知识库中存在 MCP 挂载，每轮构建 Agent 时均合并对应 MCP 工具。用户**无需**理解「先检索再出工具」顺序。 |
| 提案 B（备选） | 若产品改为「仅在某条件下挂载」，对话 UI 需增加可解释性（例如工具栏状态「检索已触发外部工具」），**本期不设计**该解释文案；定稿为 B 时须单开修订。 |

**与 README 开放问题 #1 对齐**：已在 `design-spec.md` 第一节明示采用 **提案 A** 为默认假设。

---

## 3. 用户可感知边界（默认「低调」）

### 3.1 常态（成功路径）

- **不默认展示**「本轮已加载 MCP 列表」长文案或每条消息脚注。
- **可选增强（非必须）**：助手选择器或会话标题旁 **极简图标**（如小云/插头）表示「当前助手关联外部工具」，`Tooltip` 一句「可能使用您配置的 MCP」；**默认关闭亦可**，以免干扰现有聊天视觉层级。

### 3.2 部分 MCP 失败（AC-C3）

- **不阻塞**主模型文本生成。
- **设计默认假设**（README 开放问题 #7）：同一轮用户发送后，若检测到部分 MCP 不可用，**至多一次** `message` / 轻量 Toast：**「部分外部工具暂不可用，对话仍可继续。」**（文案见 `copy-and-interaction.md`）
- **全部失败**：等价无 MCP，**不强制** Toast（避免刷屏）；若需与「部分失败」区分，可用 debug 或后续「展开详情」再暴露。

### 3.3 工具调用可观测性（与产品 #7）

- **详细列表 / 逐工具失败原因**：优先进入 **服务端日志** 与既有 **ToolTrace** 管道（PRD 提及 `onToolEvent` 方向）；**不在首版对话气泡内**展开堆栈。
- 若未来产品要求「消息内脚注」，在助手消息底部增加 **可折叠** `Collapse`「技术详情」，本期**不画死**该组件，仅保留扩展位。

---

## 4. 与「无知识库 / 无 MCP」兼容（AC-C2）

- 助手未绑定知识库，或所有绑定知识库均未挂载 MCP：**零 MCP tools**，界面与改造前一致，无新增空态。

---

## 5. 非助手会话（README 开放问题 #6）

- **设计默认假设**：本期 **不支持**「仅会话级选知识库」；无 `assistantId` 的会话 **不注入** 知识库 MCP。对话页 **不新增**「为本会话单独挂载知识库/MCP」的配置入口。

---

## 6. `enabled = false` 的 MCP（开放问题 #4）

- **运行时**：不计入可用 tools（与 US-A2 一致）。
- **UI**：不在对话侧重复警告（避免与知识库编辑页 Alert 重复）；若会话创建后用户于后台停用全部 MCP，行为退化为 AC-C2，可静默。

---
