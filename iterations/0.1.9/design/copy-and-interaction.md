# 文案与交互级别 — MCP × 知识库 × 对话（version 0.1.9）

与 `design-spec.md` 中「设计默认假设」一致；以下为建议**中文文案**与 **Toast / Modal** 使用层级，供前端与产品微调。

---

## 1. 全局原则

- **控制台**：操作结果以 **`App.useApp().message`** 为主；阻断性、需选择的用 **`Modal`**。
- **对话页**：尽量少打断；**同一用户发送轮次**内同类 Toast **合并为一次**（若技术可实现）。

---

## 2. MCP 管理页（`/console/mcp`）

| 场景 | 级别 | 建议文案 |
| --- | --- | --- |
| 保存成功 | `message.success` | 「已保存」 |
| 保存失败 | `message.error` | 服务端 `error.message` 或「保存失败，请稍后重试」 |
| 测试连接成功 | `message.success` | 「连接成功」 |
| 测试连接失败 | `message.error` | 「连接失败：{摘要}」 |
| 测试进行中 | 按钮 `loading` | 按钮旁无需长文案 |
| 删除确认 | `Modal.confirm` | 标题「确定删除该 MCP？」正文「删除后无法恢复。若仍被知识库引用将无法删除。」 |
| 删除被引用拦截 | `Modal.warning` | 标题「无法删除」正文「该 MCP 仍被以下知识库引用，请先解除挂载后再删除。」 |
| 删除成功 | `message.success` | 「已删除」 |

---

## 3. 知识库 — MCP 挂载

| 场景 | 级别 | 建议文案 |
| --- | --- | --- |
| 无 MCP 可挂 | `Alert` info | 「您还没有可用的 MCP，请先到 MCP 管理添加。」 |
| 含停用仍挂载（保存前） | `Alert` warning | 「部分已挂载的 MCP 已停用，对话中不会加载这些 MCP。」 |
| 保存知识库成功 | 沿用现有成功 Toast | — |
| 挂载非法 id | `message.error` | 通用错误，不暗示资源是否存在 |

---

## 4. 对话页

| 场景 | 级别 | 建议文案 |
| --- | --- | --- |
| 部分 MCP 失败（设计默认） | `message.warning` 或轻 Toast | 「部分外部工具暂不可用，对话仍可继续。」 |
| 全部 MCP 失败 | 无 Toast（默认） | — |

---

## 5. 可访问性补充

- `Modal` / `Modal.confirm` 确保 `title` 与 `content` 完整，避免仅用图标表达删除危险。
- 错误摘要来自服务端时应已脱敏；前端不再拼接原始 `error.stack`。
