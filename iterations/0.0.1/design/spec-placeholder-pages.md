# 0.0.1 四页面占位逐页规范

## 1. 说明

- 本文档用于落地四页面静态占位的逐页设计规范。
- 所有页面均只允许静态展示，不定义行为逻辑与数据流程。
- 页面标题与主占位文案必须严格使用需求文案，不做语义改写。

## 2. 首页（Home）

### 2.1 固定文案

- 页面标题：`首页`
- 占位文案：`首页占位中：功能模块将在后续迭代中逐步接入。`

### 2.2 区块结构

1. 顶部栏占位（项目名/品牌占位）
2. 页面标题区（H1）
3. 主占位卡片（主文案 + 辅助说明）
4. 可选次级占位区（2-3 个模块空白框，仅表达未来扩展位）

### 2.3 组件占位建议

- `PageHeaderPlaceholder`：显示页面标题。
- `EmptyStateCard`：显示首页占位文案与简短辅助提示。
- `ModuleBlockPlaceholder`：静态块级容器，不含点击或加载行为。

## 3. 登录页（Login）

### 3.1 固定文案

- 页面标题：`登录`
- 占位文案：`登录页占位中：认证流程与表单校验暂未启用。`

### 3.2 区块结构

1. 顶部栏占位（可简化）
2. 页面标题区（H1）
3. 登录卡片占位（输入框外观占位 + 按钮外观占位）
4. 占位提示区（主文案）

### 3.3 组件占位建议

- `PageHeaderPlaceholder`：显示“登录”标题。
- `FormShellPlaceholder`：仅绘制输入框与按钮外观，不支持输入提交。
- `NoticePlaceholder`：承载登录页占位文案。

## 4. 对话页（Chat）

### 4.1 固定文案

- 页面标题：`对话`
- 占位文案：`对话页占位中：消息交互与模型调用将在后续版本提供。`

### 4.2 区块结构

1. 顶部栏占位
2. 页面标题区（H1）
3. 对话内容区占位（消息气泡骨架外观）
4. 输入区占位（输入框与发送按钮外观，纯静态）
5. 占位提示区（主文案）

### 4.3 组件占位建议

- `PageHeaderPlaceholder`：显示“对话”标题。
- `ChatListShellPlaceholder`：固定数量消息占位条，不含滚动逻辑要求。
- `ChatInputShellPlaceholder`：输入栏外观占位，不可编辑、不可发送。
- `NoticePlaceholder`：承载对话页占位文案。

## 5. 控制台（Console）

### 5.1 固定文案

- 页面标题：`控制台`
- 占位文案：`控制台占位中：配置管理与数据看板功能尚未开放。`

### 5.2 区块结构

1. 顶部栏占位
2. 页面标题区（H1）
3. 统计区块占位（卡片网格外观）
4. 配置区块占位（列表或表单外观占位）
5. 占位提示区（主文案）

### 5.3 组件占位建议

- `PageHeaderPlaceholder`：显示“控制台”标题。
- `StatCardPlaceholder`：多个并列卡片，仅展示框架外观。
- `PanelShellPlaceholder`：配置区域容器外观，不含读写入口。
- `NoticePlaceholder`：承载控制台占位文案。

## 6. 统一样式补充（四页面通用）

- 主标题上方与顶部栏间距建议 24-32px。
- 标题与主文案间距建议 12-16px。
- 主占位卡片最小高度建议 160-220px，保证页面视觉稳定。
- 空白占位块可使用虚线边框或浅底色，但风格必须全站一致。

## 7. 禁止项再确认（实现边界）

- 不实现页面跳转逻辑。
- 不实现按钮点击逻辑。
- 不实现表单校验、提交、错误反馈逻辑。
- 不实现消息收发、历史记录、上下文处理。
- 不实现配置保存、数据查询、图表真实渲染。
- 不实现接口请求、鉴权、状态管理、数据持久化。

## 8. 后续开发衔接说明（仅静态壳层）

- 前端阶段仅根据本规范搭建静态页面壳层与占位组件。
- 服务端阶段在 `0.0.1` 不需对接任何接口给这四页面。
- 若要引入交互或数据能力，需在后续版本重新立项并更新需求与设计文档。
