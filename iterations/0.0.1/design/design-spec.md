# 0.0.1 设计说明（静态占位）

## 1. 文档目标与范围

- 本文档服务于 `version=0.0.1` 的阶段 2（设计）交付。
- 本迭代只定义四个页面的静态壳层与占位呈现：`首页`、`登录页`、`对话页`、`控制台`。
- 本文档不包含业务逻辑、接口流转、鉴权流程、状态机、数据库与数据读写方案。

## 2. 信息架构（IA）

### 2.1 页面清单（固定）

1. 首页（Home）
2. 登录页（Login）
3. 对话页（Chat）
4. 控制台（Console）

### 2.2 结构关系（仅表达存在性）

- 四个页面为并列一级页面，不在本迭代定义功能级跳转规则。
- 可在页面头部保留统一导航占位区，用于后续迭代接入真实导航信息。
- 当前仅强调「页面存在」与「文案一致」，不强调可用流程闭环。

## 3. 全局页面壳层规范

### 3.1 通用布局

- 页面容器建议采用三段结构：
  1) 顶部区域（品牌/项目名占位）
  2) 主内容区域（页面标题 + 占位说明）
  3) 底部辅助区域（可选的说明文本占位）
- 主内容区域建议采用居中布局，保证在桌面与移动端均易读。
- 每页至少保留一个「主占位卡片」用于承载页面说明。

### 3.2 视觉层级

- H1 页面标题：用于明确当前页面身份（必须使用需求固定标题）。
- 主文案：用于说明页面尚处占位阶段（必须使用需求固定占位文案）。
- 辅助文案：用于补充“后续迭代接入”的提示，可统一风格但不新增功能承诺。

## 4. 视觉与排版规范（基础）

### 4.1 字体与字号建议

- 字体：使用系统默认无衬线字体栈，降低初始化成本。
- 标题建议：28-32px（桌面），24-28px（平板），20-24px（移动）。
- 正文建议：14-16px，行高 1.5-1.7。
- 辅助文本建议：12-14px，颜色对比低于正文但满足可读性。

### 4.2 间距与栅格建议

- 页面左右安全边距：桌面 48px，平板 32px，移动 16px。
- 区块间距：24px（大区块）/ 16px（中区块）/ 8px（小元素）。
- 占位卡片圆角建议 10-12px，内边距建议 16-24px。

### 4.3 颜色与占位表现建议

- 背景：浅色中性底（避免强语义色）。
- 标题：高对比文本色。
- 占位卡片：中性边框 + 轻微底色区分层级。
- 图标占位：采用线性/单色图标占位风格，避免引入品牌化复杂视觉。

## 5. 响应式与设备适配建议

### 5.1 断点建议

- Desktop: `>= 1200px`
- Tablet: `768px - 1199px`
- Mobile: `< 768px`

### 5.2 适配策略

- Desktop：主内容最大宽度建议 960-1120px，避免超宽阅读。
- Tablet：保持单列，减少横向分栏。
- Mobile：全部区块单列堆叠，保证标题与占位文案优先可见。
- 四页面均遵循同一适配原则，避免每页各自定义不同规则。

## 6. 可访问性（A11y）基础要求

- 页面标题结构语义化（每页一个主标题）。
- 正文与背景对比度满足基础可读性要求（建议不低于 WCAG AA 常规文本标准）。
- 占位组件若含图标，需配套文本说明，不能仅依赖图形表达。
- 键盘焦点样式与交互行为不在本迭代实现，但视觉样式应预留空间。

## 7. 与需求对应关系

- 对应 `FR-001`：固定并仅保留四页面。
- 对应 `FR-002 ~ FR-005`：每页必须使用固定标题与固定占位文案。
- 对应 `FR-006`：全局禁止逻辑、接口、鉴权、状态管理、数据持久化。
- 对应 `AC-001 ~ AC-005`：通过统一布局和固定文案完成静态占位验收基础。

## 8. 与后续开发衔接（严格静态）

- 本设计仅允许后续开发实现：
  - 静态页面壳层（结构、样式、排版）
  - 静态占位组件（标题区、说明区、占位卡片）
- 本设计明确不允许后续在 `0.0.1` 接入：
  - 任何业务逻辑与事件闭环
  - 任何 API 请求与数据渲染
  - 任何鉴权与权限判定
  - 任何本地或远端状态存储
