# 前端验收清单 - version 0.0.1

以下为手工验收步骤，用于对照 **requirements.md** 中 AC 与 **FR-001～FR-006**。

## 环境

- [ ] `npm install` 无错误
- [ ] `npm run dev` 可启动，`npm run build` 可通过

## AC 与页面检查

### AC-001 / FR-001：仅四页面

- [ ] 仅存在四条用户可访问占位路由：`/`、`/login`、`/chat`、`/console`
- [ ] 无额外业务子页需求在 0.0.1 范围内被实现

### AC-002 / FR-002：首页

- [ ] 浏览器标题或页面 H1 可见「首页」
- [ ] 页面出现完整句子：`首页占位中：功能模块将在后续迭代中逐步接入。`
- [ ] 内容为静态展示，无动态数据、无业务流程

### FR-003：登录页

- [ ] H1 或主标题为「登录」
- [ ] 占位文案完整：`登录页占位中：认证流程与表单校验暂未启用。`
- [ ] 表单控件为 `readOnly`/`disabled`，无提交成功/失败逻辑、无接口请求

### FR-004：对话页

- [ ] 主标题为「对话」
- [ ] 占位文案完整：`对话页占位中：消息交互与模型调用将在后续版本提供。`
- [ ] 输入区与发送按钮不可真实发送（disabled/只读）

### FR-005：控制台

- [ ] 主标题为「控制台」
- [ ] 占位文案完整：`控制台占位中：配置管理与数据看板功能尚未开放。`
- [ ] 统计/配置区为静态框线，无可写操作、无图表数据请求

### AC-003 / FR-006 / AC-004：全局禁止项

- [ ] 打开 DevTools → Network：浏览四页时**无**业务 API 请求（本迭代不对接接口）
- [ ] 无登录态校验、无路由守卫弹窗或重定向逻辑
- [ ] 无全局/业务状态管理库使用痕迹（本实现仅为静态展示）

### 渲染策略（api-spec §3）

- [ ] 首页/登录/对话：查看页面源码或禁用 JS 时，**主文案仍应在 HTML 中**（SSR/RSC 静态输出）
- [ ] 控制台：首次进入可出现「加载中…」，随后为完整控制台占位；符合 **CSR + dynamic(ssr:false)** 预期

### 可访问性（design-spec §6，抽样）

- [ ] 每页有单一主标题层级（H1）
- [ ] 主要占位区域非纯图标传达（含文本说明）

## 回归说明

需求变更请回到 `iterations/0.0.1/product/`；体验/结构变更请对齐 `iterations/0.0.1/design/`；接口相关以未来定稿 `api-spec` 为准。
