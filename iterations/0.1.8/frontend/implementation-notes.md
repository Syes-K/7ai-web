# v0.1.8 前端实现说明（阶段 4）

## 最终范围（以实际交付为准）

- 对接 turn 流式事件与非流式快照，支持过程可见化。
- AI 回复区整合为“阶段流程 + 最终回复”一体化展示（assistant 风格）。
- 阶段支持“一级阶段 + 二级细节块”逐步展开。
- 失败/中断态样式区分，并支持重试。
- 重试成功后，旧失败卡禁用重试并弱化显示。
- 对话发送异常仅在对话流内展示，不再额外顶部提醒。

## 关键实现

1. `chat-api` 扩展：
   - turn 结构归一化；
   - 流式/非流式都支持 `retryUserMessageId`。
2. `ChatWorkspace` 引入 turn 视图模型：
   - 流式实时更新（started/delta/completed/failed）；
   - 历史回放（messages + turns 并行拉取）。
3. AI 回复流程展示：
   - 知识库增强、Tools/MCP、Skills、推理过程、摘要回调（有数据才展示）；
   - 每阶段支持二级细节展开（如命中知识库、命中理由、命中片段）。
4. 失败体验：
   - 失败轮次在对话内保留可见，不依赖顶部 toast；
   - 保留用户输入与失败信息，不丢失上下文。
5. 重试逻辑：
   - 按原问题重试，不新增用户消息；
   - 若同问题已有后续成功轮次，旧失败卡禁用重试并提示。
6. 消息一致性：
   - 发送后乐观插入用户消息；
   - 服务端回包后去重，避免双显示。

## 兼容与降级

- 历史会话（无 turn）仍可正常阅读消息。
- 快照字段缺失时，阶段按可用字段降级展示。
- 旧 SSE 事件（`assistant_delta`/`assistant_done`/`error`）保持兼容。

## 验证清单

1. 正常流式：阶段与最终回复连续展示，turn 终态冻结。
2. 模型失败：对话中可见失败块，可点击重试。
3. 重试成功：旧失败卡禁用重试并弱化。
4. 重试不新增用户消息：同一问题仅保留原用户输入。
5. 知识库未命中：不出现“根据知识库”误导表述。
