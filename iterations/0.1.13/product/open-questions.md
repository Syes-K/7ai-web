## 须用户确认（产品决策）

### Q1：本期范围 — 仅首页 vs 附带登录页？

| 选项 | 说明 |
| --- | --- |
| **A（推荐）** | 本期仅首页（含顶栏、metadata、页脚）完整双语；其余页面保持中文 |
| B | 首页 + 登录页 `/login` 同步双语（访客第二入口，工作量适中） |
| C | 首页 + 登录 + 聊天壳层（标题栏等）最小集 |

**默认建议**：**A**。与用户原始需求「首页支持选择语言」一致；登录页可在 `0.1.14` 跟进。

### Q3：持久化与 URL 形态

| 选项 | 说明 |
| --- | --- |
| **A（推荐）** | Locale 路径前缀：`/zh/...`、`/en/...`（或 `/` 重定向到默认 locale）；cookie 同步 |
| B | URL 不变，仅 cookie 存储 locale |
| C | URL query：`?lang=en`（一般不推荐 SEO） |

**默认建议**：**A**（`next-intl` 常见模式，利于分享与 SEO）。需接受对 `app/` 路由结构的适度调整。

### Q4：从首页进入未翻译页面时的体验

| 选项 | 说明 |
| --- | --- |
| **A（推荐）** | 静默：目标页仍为中文，但偏好已保存，不在 UI 提示 |
| B | 顶栏全局语言选择器（本期仅在首页）— 不适用 |
| C | 目标页顶部一次性 banner：「部分页面尚未提供英文」 |
| D | 未翻译页面强制回退 `zh` 并清除 `en` 偏好 |

**默认建议**：**A** + 在 README/关于页说明（若有）；避免 banner 打扰。若用户强烈关注一致性可选 **C**。

| A | 文字：`中文` \| `EN` |

## 交由设计/开发评审（需求层不限定）

### Q7：i18n 库最终选型

- **候选**：`next-intl`（与 App Router 集成成熟）、`react-i18next` + 自建 SSR 层等。
- **需求约束**：须满足 `prd.md`「i18n 方案约束」章节；开发阶段在 `iterations/0.1.13/backend/` 或前端实现说明中记录选型理由。

### Q8：`[locale]` 路由与现有 API `/api/*` 共存

- Middleware 匹配须排除 `api`、静态资源、`_next` 等；实现阶段验证 `fetch('/api/auth/me')` 等不受 locale 前缀影响。

### Q9：antd `ConfigProvider` 接入时机

- 本期首页若语言选择器用原生/HTML 实现，可暂不动态 antd locale。
- 若使用 antd `Select`/`Dropdown`，须在首页子树注入对应 `enUS` / `zhCN`。

### Q10：翻译 key 命名与目录规范

**已确认（2026-06-10）：**

| 规则 | 内容 |
| --- | --- |
| key 语言 | **英文**（camelCase 或点分层级） |
| 顶层分组 | **`page`**（页面 UI）、**`api`**（API 用户可见 message） |
| page 子组织 | **按页面各自独立文件**（如 `page/home.json`），不合并全局大文件 |
| 语义源 locale | **`en`** 为 key 对应英文文案；`zh` 为翻译 |
| 缺失回退 | 生产环境回退 **`en`** |

详见 `prd.md`「翻译 key 组织规范」章节。

### Q11：缺失 key 与回退

**已确认**：开发环境 console 警告；**生产环境回退 `en` 文案**（见 Q10）。

### Q12：SEO：`hreflang` 与 canonical

- 若采用 Q3-A，是否在 `<head>` 输出 `hreflang` 链接 — 本期可选增强，不阻塞上线。
