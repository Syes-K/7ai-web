# 用户故事与验收标准：多语言 i18n（version 0.1.13）

本文档为 `prd.md` 的子文档，便于评审与测试用例编号。

---

## Epic A：i18n 基础架构

### US-A1 站点支持中英两种 locale

**作为** 开发者/运维  
**我想要** 项目内建立统一的 `zh` 与 `en` locale 定义及文案资源组织方式  
**以便** 各页面可渐进接入翻译而不重复造轮子  

**验收标准：**

- [ ] **AC-A1**：代码库中存在明确的 locale 类型或常量，仅包含 `zh`、`en`（或等价 `zh-CN` / `en` 映射）。
- [ ] **AC-A2**：中英文案按 **`page` / `api` 两大分组**存放；**`page` 组按页面独立文件**（本期至少 `page/home`）；首页翻译 key 可从约定源加载。
- [ ] **AC-A3**：Server Component 与 Client Component 均可消费同一套 message（无两套 JSON 分叉维护）。
- [ ] **AC-A4**：`package.json` 已添加 i18n 相关依赖，与 Next.js 15 App Router 兼容（具体包名见实现说明）。
- [ ] **AC-A8**：翻译 **key 名均为英文**；`en` locale 为 key 语义源；`page` 与 `api` 分组目录结构符合 `prd.md` 约定。

### US-A2 根文档语言属性随 locale 变化

**作为** 访客  
**我想要** 浏览器与辅助技术能识别当前页面语言  
**以便** 获得正确的朗读、拼写与 SEO 信号  

**验收标准：**

- [ ] **AC-A5**：当前 locale 为 `zh` 时，根节点 `html lang` 为 `zh-CN`（或团队约定的中文标签）。
- [ ] **AC-A6**：当前 locale 为 `en` 时，根节点 `html lang` 为 `en`。
- [ ] **AC-A7**：切换语言后，上述属性随当前页更新，无需手动刷新（若架构要求整页导航，则导航后须正确）。

## Epic B：首页语言选择

### US-B1 在首页顶栏切换语言

**作为** 访客  
**我想要** 在首页顶栏选择「中文」或「English」  
**以便** 用我熟悉的语言阅读产品介绍  

**验收标准：**

- [ ] **AC-B1**：未登录与已登录状态下，语言选择器均可在 `PunkHomeHeader` 顶栏访问（位置以设计稿为准）。
- [ ] **AC-B2**：选择另一种语言后，首页主内容区文案在单次操作内全部切换，无混合语言块（合规号码除外）。
- [ ] **AC-B3**：语言选择器展示当前激活语言，且有可辨别的选中态。
- [ ] **AC-B4**：支持键盘 Tab 聚焦与 Enter/Space 选择（或可访问的等效模式）。

### US-B2 语言偏好跨会话保持

**作为** 回访访客  
**我想要** 上次选择的语言在刷新或关闭浏览器后仍然有效  
**以便** 不必每次重新选择  

**验收标准：**

- [ ] **AC-B5**：用户显式选择 `en` 后刷新首页，仍为英文（按最终 cookie/URL 策略）。
- [ ] **AC-B6**：用户显式选择 `zh` 后刷新首页，仍为中文。
- [ ] **AC-B7**：清除站点数据后，语言恢复为 **默认 `en`**（或按 Accept-Language 检测，见 `open-questions.md` Q2）。

### US-B3 首页 metadata 随语言变化

**作为** 访客 / 搜索引擎  
**我想要** 浏览器标签页标题与描述与当前语言一致  
**以便** 正确理解页面主题  

**验收标准：**

- [ ] **AC-B8**：`zh` 下首页 `title`、`description` 为中文表述（可与现网等价或优化）。
- [ ] **AC-B9**：`en` 下首页 `title`、`description` 为英文表述。
- [ ] **AC-B10**：切换语言后 metadata 与可见文案语言一致。

## Epic C：首页文案覆盖

### US-C1 顶栏导航与账户入口双语

**作为** 访客  
**我想要** 顶栏「对话」「控制台」「登录」等入口以当前语言显示  
**以便** 理解导航含义  

**验收标准：**

- [ ] **AC-C1**：`zh` 下顶栏导航文案与现网中文一致（允许微调但语义不变）。
- [ ] **AC-C2**：`en` 下顶栏导航有对应英文（如 Chat / Console / Sign in，最终措辞以设计稿为准）。
- [ ] **AC-C3**：已登录时 `UserAvatarMenu` 在首页 variant 下若有可见菜单项，本期若已暴露文案则须纳入翻译；若仅为头像无文案，则不在范围。

### US-C2 Hero 与主内容区双语

**作为** 访客  
**我想要** 标题、副标题、描述、CTA 以当前语言展示  
**以便** 理解产品定位与下一步操作  

**验收标准：**

- [ ] **AC-C4**：主标题、副标题、描述段落在 `zh`/`en` 下均有完整翻译（允许中英 slogan 风格差异，见设计稿）。
- [ ] **AC-C5**：主 CTA「进入对话」「后台控制」在 `en` 下有对应按钮文案。
- [ ] **AC-C6**：特性列表四条（`[01]`–`[04]`）在 `en` 下均有对应翻译。

### US-C3 页脚合规信息

**作为** 访客  
**我想要** 在任意语言下仍能查看备案号与联系邮箱  
**以便** 满足合规与联系需求  

**验收标准：**

- [ ] **AC-C7**：备案号 `皖ICP备2026009633号-1` 在两种语言下均原样展示且链至 `https://beian.miit.gov.cn/`。
- [ ] **AC-C8**：邮箱 `kuangyssky@163.com` 在两种语言下均可 `mailto` 打开。
- [ ] **AC-C9**：「作者邮箱」等标签文案可翻译；`SYS://local · learning mode · no warranty` 行按设计稿决定保留英文或提供翻译。

## Epic D：跨页与扩展（本期边界）

### US-D1 从首页进入其他页面

**作为** 访客  
**我想要** 在首页选择语言后进入对话或控制台时行为可预期  
**以便** 不会对「语言突然变回中文」感到困惑  

**验收标准：**

- [ ] **AC-D1**：语言偏好已写入持久化介质，后续页面迭代可读取（即使本期 `/chat` 等仍为中文）。
- [ ] **AC-D2**：跨页行为符合 `open-questions.md` 中定稿策略（无提示 / banner / 其他），测试用例与定稿一致。
- [ ] **AC-D3**：本期不要求 `/login`、`/chat`、`/console`、`/admin` 文案翻译，但不得因 i18n 改造导致这些页面 500 或布局错乱。

### US-D2 为全站迁移预留扩展点

**作为** 开发者  
**我想要** 在后续迭代为登录页、控制台等增加翻译时只需增 key 与改组件  
**以便** 控制迁移成本  

**验收标准：**

- [ ] **AC-D4**：迭代文档或代码注释中说明「新页面接入 i18n」的步骤（在 `page/` 下新增页面 message 文件、layout、antd locale）。
- [ ] **AC-D5**：`ConsoleShell` / `AdminShell` 的 `ConfigProvider locale={zhCN}` 本期可保持不动，但架构说明中已描述如何改为动态 locale。

## Epic E：非功能

### US-E1 视觉与布局

**作为** 访客  
**我想要** 语言选择器与 Punk 首页风格协调且在手机上可用  
**以便** 切换语言时不破坏整体体验  

**验收标准：**

- [ ] **AC-E1**：桌面与移动端（窄屏）顶栏不出现严重换行重叠，主操作仍可点击。
- [ ] **AC-E2**：语言控件样式与 `PunkHomeHeader` 现有 cyan/zinc 色系协调（具体 token 由设计提供）。

### US-E2 非法 locale 处理

**作为** 访客  
**我想要** 在 URL 或参数被篡改时仍能看到合理页面  
**以便** 避免白屏或裸错误  

**验收标准：**

- [ ] **AC-E3**：访问不支持的 locale 时，行为符合定稿（404 / 重定向默认 **`en`** / 其他），且无未捕获异常。
