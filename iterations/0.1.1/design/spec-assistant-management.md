## 2. 控制台 · 助手管理（`/console/assistants`）

### 2.1 页头与说明

- **标题**：`PageContainer` `ghost`，标题 **「助手管理」**（与菜单一致）。
- **Alert（info）**：说明本页列表包含 **本人创建的助手** 与 **全站系统助手**；系统助手 **不可在控制台编辑或删除**，仅供查看与后续在对话中选用（若本期实现「跳转对话」见 §3.4）。

### 2.2 列表（ProTable）

| 列 | 说明 |
| --- | --- |
| **类型** | `system`（或产品约定枚举）→ Tag「**系统**」，建议 **金色/强调色**（与公有模型 Tag 语义一致）；`personal` → Tag「**个人**」，默认色。 |
| **图标** | 展示 `icon` 字符串（emoji），无则显示占位「—」或默认图标占位符。 |
| **名称** | 主文案，可 `ellipsis` + `Tooltip`。 |
| **tags** | 多个小 `Tag` 展示；无则「—」。 |
| **开场白** | `ellipsis` 一行摘要；空则「—」。 |
| **更新时间** | `dayjs` 格式化，与模型列表风格一致。 |
| **操作** | **个人**：**编辑**、**删除**（`Popconfirm`）。**系统**：**查看**（打开只读抽屉或只读 Modal，见 §5.2）；**编辑/删除** **禁用**，`Tooltip` 文案：「系统助手请在管理后台维护」。 |

### 2.3 工具栏与筛选（满足 PRD 搜索）

置于表格 **上方工具栏**（`ProTable` `toolbar` 或 `search` 区），与表格请求联动：

| 控件 | 行为 |
| --- | --- |
| **名称** | `Input.Search` 或 `Input`，占位「搜索名称」；**模糊包含**（实现：前端过滤或 `keyword` 查询参数，二选一，以后端为准）。 |
| **tags** | `Select` `mode="tags"` **或** 单行文本输入后按逗号/空格拆分；语义为 **命中任一 tag**（OR）。 |
| **范围** | `Select`：**全部** / **仅个人** / **仅系统**。 |

- **刷新**：`ReloadOutlined` 按钮，行为与模型页一致。
- **新建助手**：主按钮 **「新建助手」** → 打开 **个人助手** 创建表单（§5.1）；系统助手 **不可** 在此创建。

### 2.4 分页与空态

- 默认分页大小与控制台模型页常量对齐（若已有 `*_DEFAULT_PAGE_SIZE` 则复用或新增 `CONSOLE_ASSISTANT_LIST_DEFAULT_PAGE_SIZE`）。
- 空列表：使用 `ProTable` `locale.emptyText`，文案说明可先去 **新建** 或等待管理员发布系统助手。

## 3. 控制台交互补充

### 3.1 新建 / 编辑（个人）

- 仅 **个人** 记录可走编辑；入口为工具栏「新建」与行内「编辑」。
- 表单字段见 §5.1。

### 3.2 删除（个人）

- 二次确认文案包含助手名称；成功后刷新列表。

### 3.3 系统行「查看」

- 只读展示全部字段（名称、提示词、图标、开场白、tags），底部仅 **关闭**；无保存按钮。

### 3.4 与对话的衔接（非 PRD 强制，建议占位）

- PRD **O1** 未强制会话绑定。设计侧建议：系统行可增加次要按钮 **「去对话使用」**，跳转 `/` 或聊天路径并携带 `assistantId`（具体路由以后端/前端约定为准）；若本期不开发，可省略按钮，仅保留 Alert 中说明。

## 4. 管理后台 · 系统助手管理（`/admin/assistants`）

### 4.1 页头

- **标题**：**「系统助手管理」**（与菜单一致）。
- **说明**：文案明确此处维护的是 **全站用户可见** 的系统助手，与控制台个人助手分流。

### 4.2 工具栏

- **新建系统助手**（主按钮）+ **刷新**。
- 筛选：与控制台类似（名称、tags）；**无需「个人/系统」筛选**（本页全部为系统），若实现简单可保留「仅搜索」而不展示范围筛选。

### 4.3 列表

| 列 | 说明 |
| --- | --- |
| **图标 / 名称 / tags / 开场白 / 更新时间** | 与控制台同类列一致，可无「类型」列（或固定展示「系统」Tag 以强化心智）。 |
| **操作** | **编辑**、**删除**（`Popconfirm`）；无禁用逻辑（管理员全权）。 |

### 4.4 新建 / 编辑

- 使用与个人相同的表单结构（§5.1），标题区分 **新建系统助手** / **编辑系统助手**。
- 保存后列表刷新；错误提示与既有 `parseApiError` 模式一致。

## 5. 表单字段（双端共用组件建议）

### 5.1 字段与控件

| 字段　　　　　　　| 组件　　　　　　　　　 | 校验与说明　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　|
| -------------------| ------------------------| ---------------------------------------------------------------------------------------------------------------------|
| **名称**　　　　　| `Input`　　　　　　　　| 必填；`trim`；最大长度建议 64（实现可配置）。　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　 |
| **提示词**　　　　| `Input.TextArea`　　　 | 必填；`showCount`；行数 6–10；最大长度建议 20000 或按后端约束。　　　　　　　　　　　　　　　　　　　　　　　　　　 |
| **图标（emoji）** | `Input`　　　　　　　　| 选填；`maxLength` 建议 **16**；占位提示示例「例如 🤖」；仅作展示字符串，不做图片上传。　　　　　　　　　　　　　　　|
| **开场白**　　　　| `Input.TextArea`　　　 | 选填；行数 3–5；最大长度建议 2000。　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　　 |
| **tags**　　　　　| `Select` `mode="tags"` | 选填；用户可输入新标签；保存前 **trim**、去空、去重（大小写规则：建议统一小写存储或原样，由后端定案，前后端一致）。 |

### 5.2 只读态（控制台查看系统助手）

- 与编辑表单相同布局，全部 `disabled` 或使用 `Descriptions`；提示词与开场白使用只读多行文本样式。

## 6. 权限与异常

| 场景 | 设计表现 |
| --- | --- |
| 未登录访问控制台/管理 API | 跳转登录，`redirect` 含当前路径（与模型页一致）。 |
| 非管理员访问管理 API | **403**，前端 `message.error` 或跳转 `console?notice=admin_forbidden`（与项目既有 admin 行为一致）。 |
| 控制台用户篡改系统助手 id 调写接口 | 后端拒绝；前端以通用错误提示展示。 |
