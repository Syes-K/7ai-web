## 1. 背景与目标

### 1.1 背景

- 管理后台与用户控制台已预留 **助手管理** 入口，当前为占位页，需落地完整能力。
- 产品需要：**运营在管理后台维护全站可选用的「系统助手」**；**登录用户在控制台维护「个人助手」**；用户在控制台列表中既能管理自己的助手，也能看到并选用系统提供的助手，且二者在界面与数据上可区分。

### 1.2 目标

1. **双端一致交付**：**系统助手**（管理后台）与 **个人控制台助手**（控制台）均提供可用的列表、创建、编辑、删除（在权限允许范围内）能力。
2. **统一列表体验**：个人控制台助手列表 **同时展示**「本人个人助手」与「系统（公有）助手」；系统助手在个人侧 **不可改删**（仅查看或选用，具体交互以设计为准，见成功标准）。
3. **类型可识别**：任意列表或详情中可明确区分 **个人** 与 **公有（系统）** 助手。
4. **字段完整**：助手具备约定的主属性，并支持在列表侧按约定维度检索。

### 1.3 成功标准（验收）

| # | 标准 |
| --- | --- |
| AC1 | 管理后台可对 **系统助手** 做增删改查；新建即为系统侧（全站可见，语义上等价「公有」）。 |
| AC2 | 控制台用户可对 **个人助手** 做增删改查；**不能**编辑或删除系统助手（可只读展示 + 选用入口，与实现一致）。 |
| AC3 | 控制台助手列表 **同时出现** 系统助手与个人助手；每条可识别类型（个人 / 系统）。 |
| AC4 | 助手字段包含：**名称**、**提示词**、**图标（emoji 文案）**、**开场白**、**tags**（多个文本）；创建/编辑表单可维护；列表可展示。 |
| AC5 | 列表支持 **按名称**、**按 tags**、**按个人或公有（系统）** 的搜索或筛选（可组合；具体为实时过滤或请求参数由设计/实现定，但用户可完成上述三类条件）。 |

## 2. 功能范围

### 2.1 本期要做（In Scope）

| # | 能力 | 说明 |
| --- | --- | --- |
| F1 | **数据类型区分** | 助手记录需区分 **个人**（归属当前登录用户）与 **系统/公有**（全站统一维护）；API 与 UI 均体现该区分。 |
| F2 | **管理后台：系统助手** | 在 `/admin/assistants` 提供系统助手列表与表单；仅管理员可写；普通用户不可通过控制台 API 篡改系统助手。 |
| F3 | **控制台：个人 + 系统同列表** | 在 `/console/assistants` 列表合并展示；个人助手可操作（编辑/删除），系统助手按 AC2 限制写操作。 |
| F4 | **字段模型** | 见第 3 节「助手属性」；tags 为 **多值文本**（字符串数组或等价结构），列表与搜索可作用于 tags。 |
| F5 | **搜索与筛选** | 至少支持：名称（模糊或包含匹配）、tags（命中任一 tag 或包含关系，以实现为准）、类型（仅个人 / 仅系统 / 全部）。 |

### 2.2 本期不做或延后（Out of Scope / 待定）

| # | 不做 / 待定 | 说明 |
| --- | --- | --- |
| O1 | 助手与对话会话的绑定策略 | 若需在聊天页默认选中某助手、会话级持久化等，可在本版仅预留选用结果或另开迭代说明。 |
| O2 | 多语言 UI | 界面语言跟随现有控制台/管理后台，不单独做 i18n。 |
| O3 | 图标超出 emoji 文案 | 本期图标以 **emoji 字符串** 为准；若未来扩展上传图片，另立需求。 |
| O4 | 系统助手的版本历史、发布审批 | 非本期必选。 |
| O5 | 按组织/租户隔离系统助手 | 当前假设系统助手 **全站** 可见；与 `0.0.10` 公有模型范围一致。 |

## 3. 助手属性（产品定义）

| 字段 | 类型（产品语义） | 必填 | 说明 |
| --- | --- | --- | --- |
| 名称 | 短文本 | 是 | 列表主标题；用于搜索。 |
| 提示词 | 长文本 | 是 | 模型 system / 行为说明类正文；具体对接聊天链路由实现阶段定义。 |
| 图标 | 短文本（emoji） | 否 | 展示用；限制为可输入的 emoji 或短字符串（避免过长破坏布局，长度上限由设计/校验约定）。 |
| 开场白 | 文本 | 否 | 用户进入对话时展示的欢迎语或首条引导；可为空则用产品默认行为。 |
| tags | 多个文本 | 否 | 零或多个标签字符串；用于分类与搜索；去重与大小写规则由实现约定（建议 trim、去空、列表内去重）。 |

**类型（个人 / 系统）**：不作为用户手填字段；由创建入口决定（控制台创建 → 个人；管理后台创建 → 系统）。

## 4. 用户故事（摘要）

| 角色 | 诉求 |
| --- | --- |
| **管理员** | 在管理后台维护系统助手库，配置名称、提示词、图标、开场白与 tags，供全站用户使用。 |
| **普通用户** | 在控制台查看「我的助手」与「系统助手」，创建与编辑自己的助手；在列表中搜索、按类型筛选；选用系统助手时不误以为自己可改系统配置。 |

---

## 5. 依赖与假设

- **鉴权**：管理后台接口沿用现有管理员鉴权（如 `withAdminApi`）；控制台接口需登录用户，且个人助手仅本人可写。
- **路由**：沿用已有菜单项 `admin-menu` / `console-menu` 中的助手入口，不强制改名（文案可微调以与设计一致）。
- **与聊天模块**：若当前聊天已存在「默认助手」或占位逻辑，本需求交付后应能挂载到真实助手数据（具体对接在设计与后端阶段明确）。
